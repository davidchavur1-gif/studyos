Administración de Discos — Conceptos
▼
ConceptoDefinición
DiscoDispositivo físico de almacenamiento (HDD, SSD, etc.)
ParticiónDivisión lógica del disco que actúa como disco independiente
Sistema de archivosEstructura que organiza los datos dentro de una partición. ext4 es el más usado en Linux
Partición primariaMBR: máx. 4 primarias · GPT: máx. 128 primarias
Partición extendidaContenedor para crear más particiones en MBR (solo en MBR)
Partición lógicaSe crea dentro de la extendida. Solo en MBR. Hasta 23 lógicas (Linux suele permitir 15 en total)
/dev/sdaPrimer disco duro SATA/SCSI
/dev/sda1Primera partición del primer disco
/dev/sr0Unidad de CD/DVD

Gestión de Particiones con parted
▼
  • Requisitos para usar una partición: 1) Tabla de particiones en el disco (MBR o GPT) · 2) Formatear con sistema de archivos (ext4)
# Crear tabla de particiones parted /dev/sdX mklabel msdos # → MBR parted /dev/sdX mklabel gpt # → GPT # Crear partición (1MiB = inicio, 500MiB = fin) parted /dev/sdX mkpart primary ext4 1MiB 500MiB parted /dev/sdX mkpart primary ext4 1MiB 10% # Segunda partición (el Start = End de la anterior) parted /dev/sdX mkpart primary ext4 1048MiB 2000MiB # Eliminar partición nº 1 parted /dev/sdX rm 1 # Redimensionar partición nº 2 hasta 800MiB (solo el End) parted /dev/sdX resizepart 2 800MiB # Formatear partición con ext4 mkfs.ext4 /dev/sdX2
📁
Montaje y /etc/fstab
▼
  • Montar = conectar la partición a una carpeta del sistema para leer/escribir. Desmontar NO borra datos, solo desconecta.
  • ¡OJO! Si una partición está montada, NO se puede redimensionar. Hay que desmontarla primero con umount.
# Crear punto de montaje mkdir /mnt/disco # Montar manualmente mount /dev/sdc1 /mnt/disco # Desmontar umount /mnt/disco # Montaje automático: añadir en /etc/fstab /dev/sdc1 /mnt/disco ext4 defaults 0 2 # campos: dispositivo · punto_montaje · tipo · opciones · dump · pass # dump=0 → sin copia de seguridad con dump # pass=1 (raíz), pass=2 (resto), pass=0 (sin comprobación)
🛡️
RAID por Software en Linux — mdadm
▼
  • mdadm = herramienta para gestionar RAID por software en Linux.
  • Importante: los discos para RAID deben estar sin particionar (sin tabla de particiones). Usar discos completos: /dev/sdb, /dev/sdc... NO /dev/sdb1.
  • El RAID se crea sobre los discos completos, luego se formatea el volumen RAID (/dev/md0).
## PASO 1 — Limpiar tabla de particiones de los discos wipefs -a /dev/sdX ## PASO 2 — Crear el RAID (ejemplos por nivel) # RAID 0 (2 discos) mdadm --create --verbose /dev/md0 --level=0 --raid-devices=2 /dev/sdc /dev/sdd # RAID 1 (2 discos) mdadm --create --verbose /dev/md0 --level=1 --raid-devices=2 /dev/sdc /dev/sdd # RAID 5 (mín. 3 discos) mdadm --create --verbose /dev/md0 --level=5 --raid-devices=3 /dev/sdb /dev/sdc /dev/sdd # RAID 10 (mín. 4 discos) mdadm --create --verbose /dev/md0 --level=10 --raid-devices=4 /dev/sdb /dev/sdc /dev/sdd /dev/sde ## PASO 3 — Formatear el volumen RAID mkfs.ext4 /dev/md0 ## PASO 4 — Montar (opcional) mount /dev/md0 /mnt/raid ## Ver estado del RAID cat /proc/mdstat # vista general mdadm --detail /dev/md0 # info completa ## Simular fallo y recuperar (RAID 1) mdadm /dev/md0 --fail /dev/sdb # marca como roto mdadm /dev/md0 --remove /dev/sdb # lo quita mdadm /dev/md0 --add /dev/sdb # añade disco nuevo ## Eliminar RAID completamente umount /dev/md0 mdadm --stop /dev/md0 mdadm --remove /dev/md0 mdadm --zero-superblock /dev/sdc mdadm --zero-superblock /dev/sdd wipefs -a /dev/sdc wipefs -a /dev/sdd
Parámetro mdadmSignificado
--createCrea nuevo dispositivo RAID
--verboseMuestra detalles del proceso
/dev/md0Nombre del dispositivo RAID resultante
--level=NNivel RAID: 0, 1, 5, 10...
--raid-devices=NNúmero de discos que forman el RAID
RAID 5 — Nota sobre capacidad con discos distintos:
  • Discos distintos en RAID 5: el sistema iguala todos al tamaño del disco más pequeño. El espacio sobrante en los más grandes no se aprovecha. ⚠️ Usar siempre discos del mismo tamaño y características.

LVM — Logical Volume Manager
▼
  • LVM = sistema de gestión de volúmenes que combina varios discos/particiones en un único espacio lógico redimensionable. Ideal cuando se necesita flexibilidad.

ComponenteQué esComando creación
PV (Physical Volume)Disco o partición convertida en volumen físico para LVMpvcreate
VG (Volume Group)Conjunto de uno o más PV que forman un grupo lógicovgcreate
LV (Logical Volume)Volumen lógico creado dentro de un VG, donde se crea el sistema de archivoslvcreate
## LVM — Crear volúmenes paso a paso # 1. Convertir particiones en volúmenes físicos (PV) pvcreate /dev/sdb1 /dev/sdc1 # 2. Crear grupo de volúmenes (VG) vgcreate grupo1 /dev/sdb1 /dev/sdc1 # 3. Crear volumen lógico (LV) de 5GB lvcreate -L 5G -n volumen1 grupo1 # 4. Formatear el volumen lógico mkfs.ext4 /dev/grupo1/volumen1 # 5. Montar el volumen mount /dev/grupo1/volumen1 /mnt/lvm

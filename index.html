🖥️
Definiciones Clave
▼
ConceptoDefinición
Sistema OperativoSoftware que gestiona y coordina hardware y software de aplicación para que el usuario pueda usar correctamente el ordenador.
SO en RedGestiona recursos compartidos en red para acceso centralizado y seguro a datos, apps y dispositivos.
KernelNúcleo del SO. Gestiona hardware y divide la RAM en espacio protegido (kernel) y espacio de usuario.
ServiciosCapa intermedia. Gestionan: procesos, memoria, dispositivos, archivos, seguridad y red.
GUIInterfaz gráfica con ventanas e íconos. Ej: GNOME (Linux), Explorador de archivos (Windows).
CLILínea de comandos. Ej: Bash (Linux), PowerShell y CMD (Windows).
🏗️
Arquitectura Von Neumann
▼
ComponenteDescripción
CPUInterpreta datos. ALU (operaciones aritméticas/lógicas) + UC (interpreta instrucciones).
RAMMemoria volátil principal. Almacena datos y programas en ejecución.
Unidades EntradaReciben datos del exterior. Ej: ratón, teclado, tarjeta de red.
Unidades SalidaDevuelven resultado de operaciones. Ej: impresora, monitor.
📊
Clasificación de SO
▼
CriterioTipos y ejemplos
UsuariosMonousuario 1 usuario (Win 95) · Multiusuario varios simultáneos (Linux, Win 11)
TareasMonotarea 1 proceso (MS-DOS) · Multitarea varios (Win 11, Linux, macOS)
Acceso a redMonopuesto 1 equipo · En red servidor+clientes propios SO · Distribuido mismo SO en varios equipos

Instalación de SO
▼
  • Código abierto — Código fuente público y modificable. Ej: Linux.
  • Código cerrado — Propietario, código no accesible. Ej: Windows.
  • Instalación atendida — Usuario responde preguntas durante la instalación.
  • Instalación desatendida — Archivo de respuesta por USB. Sin presencia del usuario.
  • Instalación limpia — Sin SO previo. Parte de cero en partición vacía.
  • Actualización — Con SO previo instalado. Conserva datos y apps.
  • VirtualBox — Máquinas virtuales. Varios SO simultáneos sin reiniciar.
  • Rufus — Crea USB booteables desde imágenes ISO.

Cuentas de Usuario
▼
  • Estándar: uso cotidiano. Necesita permiso de admin para cambios que afecten al sistema o a otros usuarios.
  • Administrador: control total. Puede instalar/desinstalar SW y HW, modificar seguridad, acceder a todos los archivos. Solo usarla cuando sea necesario.
  • Invitado: acceso temporal sin cuenta permanente. No puede instalar SW/HW ni cambiar la configuración ni crear contraseña. Hay que activarla antes de usarla.
  • UPN (User Principal Name): nombre de inicio de sesión en red Windows Server. Formato: usuario@dominio.local (ej. godofredo@iesromerovargas.local).
  • Estrategias de nomenclatura: 2+2+2 letras nombre/apellido1/apellido2, o 3 nombre + 2 apellido1, o 4 nombre + ini1 + ini2 + 2 últimos DNI.
  • Tipos en AD: Usuario (persona física), Grupo (contenedor), Equipo (máquina). Las cuentas de grupo son contenedores de las otras dos.

Contraseñas y Directivas
▼
  • Buena contraseña: mínimo 6 caracteres, mayúsculas + minúsculas + número, no es el nombre ni datos personales, no es rotación del login.
  • Mejor técnica: acrónimo alfanumérico de una frase. Ej: "hasta luego Lucas" → H4st4luegoluc4S.
  • Opciones al crear usuario en AD: El usuario debe cambiar la contraseña en el siguiente inicio de sesión / El usuario no puede cambiar la contraseña / La contraseña nunca expira / La cuenta está deshabilitada.
  • Cuenta deshabilitada: se puede deshabilitar manualmente o automáticamente (bloqueo por intentos fallidos). En producción, mejor deshabilitar que eliminar.
  • Creación desde CMD: dsadd user "CN=MARGARITA,CN=USERS,DC=BIXO,DC=COM" -pwd Abcd1234
  • Comandos administración: dsmod (modificar), dsrm (eliminar), dsmove (mover), dsget (info). Formato: dsmod user "nombre_distinguido" -opcion valor

Grupos y Ámbito en AD
▼
  • Grupos de distribución: solo correo electrónico. NO para seguridad.
  • Grupos de seguridad: asignar derechos y permisos a usuarios y equipos. Más eficaz cuando está anidado.

# Crear grupo en AD desde CMD
dsadd group "CN=VIKINGOS,CN=USERS,DC=BIXO,DC=COM"

# Opciones útiles: -secgrp yes|no / -scope l|g|u
# Añadir miembros a un grupo
dsmod group "CN=VIKINGOS,..." -addmbr "CN=USUARIO1,..."
🏢
Unidades Organizativas (OU) y GPOs
▼
  • OU (Organizational Unit): contenedores lógicos en AD para organizar usuarios, grupos y equipos. Estructuran la jerarquía administrativa.
  • Usos comunes: por departamento (RRHH, IT, Finanzas), por ubicación geográfica, por función (Servidores, Estaciones de Trabajo).
  • Crear OU (PowerShell): New-ADOrganizationalUnit -Name "Departamentos" -Path "DC=prueba,DC=local"
  • GPO (Group Policy Object): vinculado a una OU para configurar restricciones de seguridad y comportamiento de usuarios y equipos.
  • Crear/Editar GPO: Herramientas → Administración de directivas de grupo → clic derecho en OU → Nueva GPO → Editar.
⚙️
Gestión de Procesos
▼
  • Proceso: programa en ejecución que requiere CPU, memoria y dispositivos E/S. El SO gestiona la planificación mediante el PCB.
  • PCB (Process Control Block): estructura de datos que representa un proceso: apuntador, estado, número de proceso, contador de programa, registros, límites de memoria, lista de archivos abiertos.
  • Estados de un proceso: Nuevo → Preparado → Ejecución → Bloqueado → Terminado.
  • Planificación expulsiva: el SO puede desalojar un proceso de la CPU antes de que termine (Round Robin, prioridades dinámicas).
  • Planificación no expulsiva: el proceso no puede ser desalojado hasta que termina (FCFS/FIFO).

# Listar procesos en CMD
tasklist
tasklist /FI "IMAGENAME eq chrome.exe"

# Matar proceso por PID o nombre
taskkill /PID 1234 /F
taskkill /IM notepad.exe /F

# Cambiar prioridad en PowerShell
(Get-Process chrome).PriorityClass = "High"

DNS en Windows Server
▼
  • DNS: Domain Name System. Traduce nombres de dominio a IPs y viceversa. Sin DNS, hay que usar IPs numéricas.
  • Zona de búsqueda directa: nombre de dominio → IP. Registros tipo A (IPv4) y AAAA (IPv6). Ej: www.empresa.local → 192.168.1.102
  • Zona de búsqueda inversa: IP → nombre de dominio. Registros tipo PTR. Ej: 192.168.1.10 → www.empresa.local. Útil para auditorías de seguridad.
  • Registro A / Host nuevo: Nombre (www), FQDN (www.prueba.local), IP (192.168.68.51). Opcional: crear registro PTR asociado.
  • nslookup: herramienta CMD para diagnosticar DNS. Búsqueda directa: nslookup www.prueba.local. Inversa: nslookup 192.168.68.51

# Pasos configurar DNS en Windows Server
# 1. Herramientas Administrativas → DNS
# 2. Nueva zona de búsqueda inversa (para PTR)
   → Primeros 3 octetos de la red: 192.168.68
# 3. En zona directa → clic derecho → Host nuevo
   → Nombre: www, IP: 192.168.68.51, marcar PTR

# Configurar cliente para usar el servidor DNS
   → TCP/IPv4 → DNS preferido: IP del servidor

# Verificar resolución
nslookup www.prueba.local
nslookup 192.168.68.51

¿Qué es RAID?
▼
  • RAID = Redundant Array of Independent Disks. Combina varios discos para formar una unidad lógica.
  • Ventajas vs disco único: mayor integridad, tolerancia a fallos, mayor rendimiento, más capacidad.
  • RAID Hardware → controladores PCI dedicados. Independiente del SO. Más rápido, mejor integridad.
  • RAID Software → implementado por el SO. Más barato pero depende de la plataforma.
  • Discos para RAID: IDE/SATA (baratos, no ideales), SCSI/SAS (servidores), SSD (mejor rendimiento I/O).

RAID 1 en Windows Server — Pasos
▼
  • 1. Server Manager → Tools → Computer Management → Storage → Disk Management
  • 2. Clic derecho sobre un disco → ver opciones: Spanned, Striped, Mirrored, RAID-5
  • 3. Seleccionar "New Mirrored Volume" → asistente Wizard
  • 4. Añadir el segundo disco con "Add" → ajustar tamaño
  • 5. Asignar letra de unidad (ej. M: para Marketing, T: para Técnico)
  • 6. Seleccionar sistema de archivos: NTFS (recomendado)
  • 7. Confirmar en resumen del Wizard → "Finish"
  • 8. Aceptar conversión de discos a dinámicos → esperar % de progreso
  • 9. Verificar estado "Healthy" en Disk Management + nueva unidad en explorador

Tolerancia a Fallos — Conceptos clave
▼
ConceptoDescripción
Creación de reflejoMantiene copias idénticas de todos los datos. Similar a RAID-1. Bidireccional o tridireccional.
Paridad (XOR)Codificación que permite reconstruir datos perdidos calculando el XOR de los demás discos.
Disco dinámicoTipo de disco Windows que permite gestión avanzada de volúmenes (necesario para RAID por software).
Mirrored VolumeNombre en Windows para RAID 1. Genera copia dinámica en tiempo real.
Hot spareDisco de repuesto en espera. Se activa automáticamente si falla otro disco.
